Objective

Build an affordable, extensible bioreactor monitoring platform capable of pharmaceutical-grade microbial culture tracking. An Arduino reads an array of environmental sensors every second and streams data over serial to a host PC, where a Python logger saves timestamped CSV files for downstream analysis.

Two pilot studies with E. coli cultures have been completed. Each study incrementally added sensors. The platform is designed to grow with the research — new sensors or actuators can be added to the same I2C bus and integrated with minimal code changes.

Hardware

The Arduino microcontroller (no real-time clock) communicates at 115,200 baud. All sensors share a single I2C bus. Absolute timestamps are added by the host PC Python logger.

Arduino Firmware

user_provided/arduino/01_stepper_motor/01_stepper_motor.ino — Written in C/C++. Reads all sensors every second, streams comma-separated rows over serial, and manages pump duty-cycle timing and stepper motor speed.

Required Arduino libraries:

Wire (built-in) Adafruit_SHT31 Adafruit_BME680 DFRobot_AS7341

Python Data Loggers

Scripts in user_provided/python/archive/ (archived copies; originals in user_provided/arduino/01_stepper_motor/) open the Arduino serial port, parse incoming CSV lines, prepend a PC timestamp and elapsed time, and write rows to a .csv log file. Terminal output shows live sensor values. Scripts handle malformed packets and Arduino startup noise gracefully.

Data Pipeline

user_provided/python/generate_study_summaries.py scans every studies/study*/ folder, reads all CSV files, normalises column names across five historical schema versions, filters bad/saturated sensor values, downsamples to ~4 000 points per study for browser performance, and writes one JS file per study to docs/js/. Running bash user_provided/makefile/build.sh re-runs the pipeline and opens the site.

Each generated JS file populates two globals:

// Per-study logging statistics
window.STUDY_SUMMARIES["study001_ecoli"] = {
  study, generated,
  timeline: {
    first, last, wall_clock_hours, total_logged_hours,
    session_count, gap_count,
    sessions: [{ file, start, end, duration_s, rows, sensors }],
    gaps:     [{ after_file, before_file, gap_s, gap_hours }]
  },
  variables: {
    sht30_temp_C: { label, unit, total_rows, valid_rows, bad_rows,
                    bad_duration_s, bad_reasons, bad_windows,
                    min, max, range, mean },
    ...
  }
};

// Per-study downsampled time-series chart data
window.STUDY_CHARTS["study001_ecoli"] = {
  original_rows, sampled_rows,
  x: ["2026-06-10 19:19:19", ...],   // shared timestamps (one array for all traces)
  groups: {
    temperature: { title, unit, traces: [{ name, color, y: [21.3, null, 22.1, ...] }] },
    humidity:    { ... },
    pressure:    { ... },
    gas:         { ... },
    light:       { ... }
  }
};

Null values in y arrays mark bad/saturated readings and session gaps — Plotly renders these as clean line breaks rather than connecting across missing data.

Web Dashboard

docs/index.html is a fully static single-page site with no build step beyond running the Python pipeline. It uses two JavaScript libraries loaded from CDN:

Plotly.js
Interactive time-series charts for each study: session timeline (Gantt-style), bad-data duration bar chart, dual-axis Temperature °C + Pressure hPa chart, and per-group charts (temperature, humidity, pressure, gas resistance, light spectrum). Also powers the drag-and-drop live CSV visualizer.
Tabulator v6
Sortable, filterable, downloadable tables for the hardware manifest (sensors, actuators, logger scripts) and the per-study variable statistics table (min / max / range / mean / bad-row count per sensor column). All tables export to CSV via a Download button.

Quick Start

# 1. Upload 01_stepper_motor.ino to the Arduino via Arduino IDE
# 2. Connect Arduino via USB — confirm serial port (e.g. /dev/ttyUSB0 or COM3)
# 3. Install dependency
pip install pyserial

# 4. Run the current full-sensor logger (from user_provided/arduino/01_stepper_motor/)
python logging_sht30_bme688_bme688_as7341.py

# 5. After a study session, move CSVs to studies/study00X_ecoli/ and rebuild the site
bash user_provided/makefile/build.sh

Future Work

Data analysis pipeline
Build Python scripts or Jupyter notebooks to visualize and analyze logged CSV data — temperature trends, gas resistance as a culture-growth proxy, spectral changes over time.
Real-time clock (RTC) integration
Add an RTC module to the Arduino so timestamps originate on the microcontroller, removing the dependency on the host PC for absolute time.
Closed-loop control
Use sensor feedback (temperature, humidity, gas resistance) to drive actuator decisions — e.g., adjust pump duty cycle or stepper speed in response to culture state.
Additional organisms and media
Extend studies beyond E. coli to other microorganisms or culture conditions. The modular sensor stack makes it straightforward to add optical density (OD600) or pH sensing.
Live web dashboard
Serve live sensor data via a local web interface so the reactor can be monitored remotely without SSH access to the logging PC.
Documentation expansion
Add wiring diagrams, a bill of materials (BOM), and assembly instructions to the docs/ folder for reproducibility.
